# SOFT37 DESIGN SYSTEM

> **Vision** — 일상에 스며드는 자연스러움
> **Direction** — Cool Minimalism (선명함 · 신뢰감 · 트렌디 · 열린 접근성)
> **Goal** — 작은 UI 요소나 로고만 보고도 "어, 이거 Soft37 거 아냐?" 하게 만드는 일관된 BI

이 문서는 Soft37의 웹사이트, 앱, 아이콘, 마케팅 자산 전반의 **단일 기준(Single Source of Truth)** 입니다.
새 화면·자산을 만들 때는 항상 이 문서의 토큰과 규칙을 따릅니다. (하드코딩 금지)

> **톤 한 줄 요약**: 차갑지만 닫혀 있지 않다. 선명하고 믿음직하되, 넉넉한 여백과 둥근 형태로 누구나 쉽게 다가온다.

## 1. 브랜드 기반

### 비전
**일상에 스며드는 자연스러움.**
사용자가 "도구를 쓴다"고 의식하지 않을 만큼, 자연스럽게 일상에 녹아드는 서비스를 만든다.

### 브랜드 성격 (4 키워드)
| 키워드 | 의미 | 디자인 표현 |
|--------|------|-------------|
| 선명함 (Clear) | 군더더기 없이 또렷하다 | 높은 대비, 정밀한 정렬, 쿨 뉴트럴 |
| 신뢰감 (Trust) | 도구로서 믿음직하다 | 일관된 시스템, 차분한 쿨 톤, 정돈된 구조 |
| 앞서감 (Forward) | 트렌드를 이끈다 | 선명한 블루 액센트, 굵은 디스플레이 타이포 |
| 열림 (Open) | 누구나 쉽게 다가온다 | 넉넉한 여백, 둥근 형태, 친근한 카피 |

### 디자인 원칙
1. **차갑되 닫지 않는다** — 쿨 톤·고대비로 신뢰를 주되, 여백과 둥근 형태로 문을 열어 둔다.
2. **하나의 시그니처를 끝까지 반복한다** — 인식성은 새로움이 아니라 반복에서 나온다.
3. **덜어내는 쪽을 택한다** — 의심스러우면 뺀다. 여백은 디자인이다.
4. **선명한 액센트는 아껴 쓴다** — 블루는 정말 중요한 곳에만. 흔해지면 신뢰가 묽어진다.
5. **조용히, 정확히 움직인다** — 튕김 없이 매끄럽게 안착한다.
6. **토큰으로만 만든다** — 모든 색·간격·곡률은 토큰을 참조한다. 임의값 금지.

## 2. 시그니처 요소 (인식의 핵심)

Soft37의 인식성은 아래 **3개의 DNA**가 모든 자산에 반복되며 만들어진다.

| DNA | 정의 | 어디에 |
|-----|------|--------|
| **ㅅㅊ 마크** | 37=삼칠=ㅅㅊ 을 형상화한 한 획의 봉우리 + 스파크 | 로고, 앱 아이콘, 파비콘, 로딩 |
| **둥근 획 끝 (round cap)** | 모든 선의 끝은 둥글게 | 아이콘, 그래프, 마크, 구분선 |
| **단일 곡률 (soft radius)** | 어디서나 같은 모서리 곡률 | 버튼, 카드, 입력창, 이미지, 모달 |

> **핵심 서사**: **37 → 삼칠 → ㅅㅊ**.
> **ㅅ** = 한 획의 봉우리. 곧은 왼다리(7=정확함) + 흐르며 말리는 오른쪽 루프(3=부드러움).
> **ㅊ** = 선명한 블루 스파크. 차가운 신뢰(blue) × 열린 흐름(곡선) — 이 대비가 Soft37이다.

### 공식 마크 (ㅅㅊ)
한 획으로 흐르는 ㅅ과 블루 스파크 ㅊ. 아래 정의가 마크의 정본(canonical)이다.
(자산: `soft37_logo_mark.svg`)

```svg
<svg viewBox="0 0 470 300" xmlns="http://www.w3.org/2000/svg">
  <!-- ㅅ : 직선 왼다리 + 오른쪽 루프 -->
  <path d="M 95 235 L 190 65 C 235 150 280 195 305 212 C 320 224 342 226 358 214 C 376 200 378 174 362 160 C 348 148 324 150 316 166 C 310 178 318 190 332 190"
        fill="none" stroke="#15181D"
        stroke-width="22" stroke-linecap="round" stroke-linejoin="round"/>
  <!-- ㅊ : 블루 스파크 -->
  <g stroke="#2F6BFF" stroke-width="11" stroke-linecap="round">
    <line x1="375" y1="82" x2="375" y2="138"/>
    <line x1="361" y1="86" x2="389" y2="134"/>
    <line x1="389" y1="86" x2="361" y2="134"/>
  </g>
</svg>
```

- **곧은 왼다리** → "7"의 정확함 / 도구적 신뢰
- **오른쪽으로 흐르며 말리는 루프** → "3"의 부드러움 / 열린 흐름
- **둥근 꼭짓점 · round cap** → 차갑지 않게, 다가오기 쉽게
- **블루 스파크(ㅊ)** → 유일한 포인트 컬러가 머무는 자리

## 3. 로고 · 심볼 시스템

### 구성
| 형태 | 용도 |
|------|------|
| **심볼** (ㅅㅊ 마크 단독) | 앱 아이콘, 파비콘, 프로필, 워터마크 |
| **로고타입** (Soft37 워드마크) | 본문 헤더, 푸터, 문서 |
| **시그니처** (심볼 + 워드마크 가로조합) | 웹사이트 헤더, 명함, 공식 자산 |

### 클리어 스페이스
심볼 높이를 `X`라 할 때, 사방으로 최소 **0.5X**의 여백을 비운다. 이 영역엔 어떤 요소도 넣지 않는다.

### 스파크(ㅊ) 색 규칙
- 기본: **Blue `#2F6BFF`**.
- 단색 인쇄·음각·아주 작은 크기 등 컬러를 못 쓰는 경우에만 ink 또는 배경색으로 통일(단색 마크) 한다.
- 그 외 스파크를 Blue 외 색으로 바꾸지 않는다.

### 컴팩트 버전 (작은 크기 전용)
- **16~48px** (파비콘, 앱 아이콘 소형, 인앱 뱃지)에서는 기본 마크의 획이 가늘어 무너진다.
- 이 구간에서는 **획 두께를 약 1.3배** 키운 *컴팩트 마크*를 사용한다. (stroke-width 22 → 28 기준)
- 24px 미만에서는 오른쪽 루프(컬) 디테일을 단순화하고, 필요 시 스파크를 생략한 ㅅ 단독을 쓴다.

### 금지 사용 (Misuse)
| ❌ 하지 말 것 |
|--------------|
| 마크를 기울이거나 회전 |
| 획 끝을 각지게 (round cap 제거) |
| 곡률·비율 임의 변형 |
| 브랜드 외 색으로 채색 (지정 팔레트 외 금지) |
| 그림자·그라데이션·외곽선 추가 |
| 저대비 배경 위에 배치 (가독 X) |

## 5. 형태 언어 (Shape)

### 모서리 곡률 (Radius)
> 따뜻·아기자기함을 덜기 위해 곡률을 한 단계 단단하게 조정(v2.0). 둥글되 과하지 않게 — 현대적·신뢰 + 열림의 균형.

| 토큰 | 값 | 용도 |
|------|-----|------|
| `radius/xs` | 6 | 태그, 칩, 작은 뱃지 |
| `radius/sm` | 8 | 작은 버튼, 인풋 내부 요소 |
| `radius/md` | 12 | **버튼 · 입력창 (기본)** |
| `radius/lg` | 16 | **카드 · 패널 (기본)** |
| `radius/xl` | 24 | 모달 · 바텀시트 · 큰 컨테이너 |
| `radius/full` | 999 | pill 버튼, 아바타, 토글 |

- **앱 아이콘**: iOS 스퀴클(superellipse) 비율 **≈22.4%** 적용.
- 한 화면 안에서 곡률을 섞지 않는다. (버튼 12 / 카드 16처럼 위계는 유지하되 임의값 금지)

### 그림자 (Elevation) — 쿨하게, 절제해서
| 토큰 | 값 |
|------|-----|
| `shadow/e1` | `0 1px 2px rgba(20,24,30,0.06)` |
| `shadow/e2` | `0 4px 12px rgba(20,24,30,0.08)` |
| `shadow/e3` | `0 12px 32px rgba(20,24,30,0.12)` |
| `focus/ring` | `0 0 0 3px rgba(47,107,255,0.30)` |

- 그림자는 옅게. 분리는 가능하면 **`ink/200` 보더**로. 네온·글로우 금지.
- 포커스는 항상 **blue 포커스 링**으로 또렷하게 (접근성 + 신뢰).

### 보더
- 기본 두께 **1px**, 색 `ink/200`. 강조 시 `ink/300`. 활성/포커스 시 `brand/blue`.
- 모든 선의 끝/모서리는 둥글게 (round join).

## 6. 타이포그래피

### 폰트
| 용도 | 폰트 |
|------|------|
| 국문 | **Pretendard** |
| 영문 · 숫자 | **Inter** |
| 코드 · 수치 모노 (선택) | **JetBrains Mono** |

> 부드러움·따뜻함이 아니라 **선명함**을 글자가 담당한다. 큰 제목은 굵고 또렷하게(앞서가는 인상), 본문은 차분·명료하게.

### 굵기
`400 Regular` · `500 Medium` · `600 SemiBold` 기본. **`700 Bold`는 display 한정** 허용.

### 타입 스케일
| 토큰 | size / line | weight | 자간 |
|------|-------------|--------|------|
| `display` | 40 / 48 | 700 | -0.02em |
| `h1` | 32 / 40 | 600 | -0.02em |
| `h2` | 26 / 34 | 600 | -0.01em |
| `h3` | 20 / 28 | 600 | -0.01em |
| `title` | 17 / 24 | 500 | 0 |
| `body` | 15 / 24 | 400 | 0 |
| `body-sm` | 13 / 20 | 400 | 0 |
| `caption` | 12 / 16 | 400 | 0 |
| `label` | 15 / 20 | 500 | 0 |

- 본문 행간은 넉넉히(1.6 전후) — 여백 = 열림.
- 숫자(가격·통계)는 Inter `tabular-nums` 사용해 정렬 안정.

## 7. 아이코노그래피

### UI 아이콘
- **라인 스타일**, 24px 그리드 기준.
- 획 두께 **1.75px**, **round cap / round join** (마크의 둥근 획과 호응).
- 채움(fill) 아이콘은 활성 상태 등 예외적으로만.
- 한 세트 톤 일관 유지 — 라인/필 혼용 금지.
- 추천 베이스 세트: Lucide 또는 Phosphor(라인). 커스텀 아이콘도 동일 규칙으로 그린다.

### 앱 아이콘
- 배경: `surface`(화이트) · `brand/blue` · `ink/900`(다크) 중 택1, 스퀴클(곡률 ≈22.4%).
- 마크: 컴팩트 ㅅㅊ 마크를 중앙, 광학 정렬. 사방 클리어 스페이스 0.5X 확보.
- 그림자·테두리·텍스트 없이 마크 단독.
- 세 가지 공식 조합:
  - **화이트 배경 + ink ㅅ + blue 스파크** (라이트, 기본)
  - **블루 배경 + 화이트 ㅅ + 화이트 스파크** (강조, 단색)
  - **잉크(니어블랙) 배경 + 화이트 ㅅ + blue 스파크** (다크, 트렌디)
- 아주 작은 사이즈에서 스파크가 뭉개지면, ㅅ 단독으로 떨어뜨린다.

## 8. 레이아웃 · 여백

`4 · 8 · 12 · 16 · 20 · 24 · 32 · 40 · 48 · 64 · 96`

| 토큰 | 값 | 용도 |
|------|-----|------|
| `space/1` | 4 | 아이콘-텍스트 간격 |
| `space/2` | 8 | 인접 요소 |
| `space/3` | 12 | 컴포넌트 내부 패딩 |
| `space/4` | 16 | **기본 간격** |
| `space/6` | 24 | 카드 패딩, 섹션 내 그룹 |
| `space/8` | 32 | 블록 사이 |
| `space/12` | 48 | 섹션 사이 |
| `space/16` | 64 | 큰 섹션 사이 |

### 그리드

- 앱: 16px safe 패딩 기준.
- **여백을 아끼지 않는다.** 빽빽함보다 숨 쉴 공간 — "열림"은 대부분 여백에서 나온다.

## 9. 모션

### 원칙
조용히, 정확히, 매끄럽게. **튕김(bounce)·과장된 스프링 금지.** 통제된 움직임이 신뢰를 만든다.

### 지속시간
| 토큰 | 값 | 용도 |
|------|-----|------|
| `motion/fast` | 120ms | hover, 작은 상태 변화 |
| `motion/base` | 200ms | **기본 전환** (버튼, 토글) |
| `motion/slow` | 280ms | 카드 등장, 패널 |
| `motion/deliberate` | 400ms | 모달, 페이지 전환 |

### 이징
| 토큰 | 곡선 | 용도 |
|------|------|------|
| `ease/out` | `cubic-bezier(0.2, 0, 0, 1)` | 등장 (기본) |
| `ease/in-out` | `cubic-bezier(0.4, 0, 0.2, 1)` | 이동 |
| `ease/in` | `cubic-bezier(0.4, 0, 1, 1)` | 퇴장 |

- 페이드 + 작은 이동(8~12px) 조합을 기본으로. 크게 날아오는 모션 지양.
- 인터랙티브 요소(링크·버튼)는 hover 시 blue 강조가 매끄럽게 전환되도록.

## 10. 보이스 & 톤

- **담백하고 자신감 있는 존댓말.** 과장·아기자기·느낌표 남발 없이, 명료하고 또렷하게.
- 차가운 비주얼을 말투의 따뜻함이 보완한다 — 차갑게 보이되 **읽으면 친근하게**. ("열림"은 카피가 담당)
- 기능을 자랑하기보다 *사용자가 할 수 있는 것*을 또렷이 안내한다.
- 짧고 명료하게. 한 문장에 한 가지.

| 상황 | ❌ | ✅ |
|------|----|----|
| CTA | "지금 바로 시작하세요!!" | "시작하기" |
| 빈 상태 | "앗! 아직 아무것도 없어요 😢" | "아직 추가된 항목이 없습니다. 첫 항목을 추가해 보세요." |
| 에러 | "오류가 발생했습니다! 다시 시도!" | "저장하지 못했습니다. 잠시 후 다시 시도해 주세요." |
| 성공 | "🎉 성공!!" | "저장되었습니다." |

### "어디서 봤는데?" 인식성 체크리스트
새 자산을 내보내기 전 확인:
- [ ] ㅅㅊ 마크가 규칙대로(곡률·round cap·스파크·클리어스페이스) 들어갔는가
- [ ] 스파크가 `brand/blue`인가 (단색 예외 외)
- [ ] 모서리 곡률이 토큰(12/16/24) 중 하나인가 — 임의값 없는가
- [ ] 모든 선의 끝이 둥근가 (round cap)
- [ ] 블루가 화면의 10% 이내인가 (액센트로만)
- [ ] 배경이 **쿨 뉴트럴**인가 (따뜻한 베이지/샌드 아님)
- [ ] 포커스 상태에 blue 포커스 링이 있는가
- [ ] 여백이 충분히 넉넉한가 (열림)
- [ ] 폰트가 Pretendard + Inter, 굵기 규칙(400/500/600, display만 700) 안인가
- [ ] 문구가 담백·자신감 있고, 아기자기하지 않은가

> 이 항목들이 모든 화면에서 지켜지면, 사용자는 로고가 없어도 Soft37을 알아본다.
